# AUTOMATION - 重复工作处理流程

## 适用范围

凡是明显会再次发生的任务，都先走这条流程，不要长期停留在一次性手工操作。

## 流程

1. 概念
   - 先描述这件事的标准过程、输入、输出和边界。
2. 原型
   - 先对 3-10 个真实项手工执行，不先写 skill 文件。
3. 评估
   - 把输出展示给用户，确认是否符合预期。
4. 编纂
   - 如果值得复用，就写成 `SKILL.md`，或扩展已有 skill。
5. Cron
   - 如果它应该自动运行，就用 `openclaw cron add` 安排。
6. 监控
   - 第一次自动运行要检查结果，再迭代。

## 规则

- 每个能力域尽量只有一个主 skill。
- 创建新 skill 前，先检查现有 skill 是否已经覆盖。
- 如果已有 skill 覆盖，就扩展它，不要新建重叠 skill。
- 不要把“我记住了下次再做”当成最终交付。
- 如果用户再次要求同类工作，目标不是重复手工执行，而是把它收束成 skill 或 cron。
